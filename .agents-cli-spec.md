# Agent Spec

## Overview

Extend local-agent's global curated memory so it retains relevant, reusable
facts provided by authorized users about people, systems, projects, decisions,
preferences, and operational state. Explicit requests to remember or save a
fact receive priority, but persistence remains asynchronous and curator-driven.

## Example Use Cases

- "Mi nombre es Dauno y soy el creador de local-agent" creates or updates a
  curated person/role topic.
- "Recuerda que producción usa PostgreSQL 16" creates or updates a system
  topic.
- "Prefiero respuestas en español" records a durable preference when it is
  relevant to future replies.

## Tools Required

- Existing Slack Socket Mode events.
- Existing SQLite memory store and FTS recall.
- Existing curator LLM for structured patch proposals.

## Constraints & Safety Rules

- Memory remains global to authorized users in this local-agent project.
- Store curated facts, not raw transcripts.
- Never persist credentials, secrets, authentication material, or prompt,
  tool, authorization, and policy instructions.
- Treat recalled memory as untrusted reference material.
- Preserve evidence, revisions, deletion behavior, budgets, and idempotency.
- Do not guarantee persistence of irrelevant, contradictory, or unsupported
  claims; retain self-declared facts with attribution when reusable.

## Success Criteria

- A self-declared identity or role is curated after a successful exchange.
- An explicit memory request makes a relevant supplied fact eligible for
  curation.
- A later cross-thread question recalls the curated fact.
- Safety validation continues to reject prohibited content.
